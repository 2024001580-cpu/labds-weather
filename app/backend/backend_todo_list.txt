* POST user                                           [ ]
Rota de cadastro do usuário. Deve fazer todas as validações necessárias.
Não deve permitir caso os headers já estejam preenchidos com dados de autenticação.
* GET user                                            [ ]
Retorna dados do usuário.
Autenticar.
* PUT user                                            [ ]
Atualiza dados do usuário.
Recebe dados pelo corpo da requisição, atualiza somente os que não estiverem vazios.
Validações obrigatórias para cada campo.
Autenticar.
* DELETE user                                         [ ]
Exclui o usuário.
Autenticar.

* POST user/authenticate                              [ ]
Definir como será feita a autenticação. Mas penso em um token gerado a partir do email, senha e horário de acesso.
Esse token será recebido pelos headers de todas as requisições para as quais for necessário.
Será validado para cada ação que precisar.

** Logout poderá ser feito pelo front, apenas apagando dados do cache que seriam enviados pelo header.
